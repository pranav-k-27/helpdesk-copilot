Dual-Pipeline Request Flow

📚 RAG Pipeline — "How-to / Policy" Queries

1
NLQ received & classified as How-toe.g. "How do I reset a user's VPN access?"
2
Guardrails check (PII, injection)Clean query passed forward
3
Hybrid retrieval — Top-10 chunksDense vector search + BM25 keyword search merged
4
Cross-encoder reranker → Top-3Best chunks selected by relevance score
5
LLM generates answer from context onlyCitations included. If confidence low → fallback
6
Response logged to observability layerFull trace: query → docs → answer → tokens

📊 NLQ→SQL Pipeline — "Analytics / Insight" Queries

1
NLQ received & classified as Analyticse.g. "Which ticket categories breach SLA most?"
2
Schema context retrieved from vector storeRelevant table/column metadata injected into prompt
3
LLM generates SQL querySafe, parameterised query with validation
4
SQL executed on ticket databaseReturns structured result set
5
LLM narrates results in plain EnglishWith trend highlights and SLA risk flags
6
Response logged to observability layerSQL, results, and narration all audited
